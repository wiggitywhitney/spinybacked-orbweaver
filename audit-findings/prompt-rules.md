## Summary

### High-severity ambiguities

These five are M3's work list — fixes are rewrites of existing wording only, no rule redesign required:

1. **COV-004 process.exit() exception missing COV-001 override** — causes oscillation on any exported async function that calls process.exit(). Fix: add "and is not a COV-001 entry point" qualifier to COV-004's exception clause.

2. **Ratio backstop `~20%` undefined action** — causes the LLM to choose different fallback behaviors (skip file / instrument only entry points / instrument all and warn) across runs. Fix: replace `~20%` with `20%` and specify the action (instrument COV-001 and COV-002 only).

3. **CDQ-006 root span exemption undefined** — causes inconsistent `isRecording()` guard application across runs. Fix: replace "root spans or spans created at the entry point of a traced operation" with "spans on COV-001 entry points."

4. **Return-value capture exception scope** — the mandatory `await` in the pattern example implies async-only, but the prohibition is stated only for object literals, not synchronous expressions. Causes unnecessary `await` on synchronous captures in some runs. Fix: add explicit "async expressions only" restriction and clarify whether `await` is required.

5. **Namespace prefix inference in mixed registry** — "first segment of existing registered attribute names" returns multiple namespaces when the registry contains both OTel semconv and org-specific keys. Fix: add a clarifying sentence specifying that org-specific namespace takes precedence over imported OTel convention namespaces.

### Systemic patterns

The ambiguities are not isolated — they share a structural root cause: **rules were added incrementally without updating all locations that state the same constraint**. The RST-006 / COV-001 / COV-004 triangle (all three state or reference the process.exit() rule with different completeness) is the clearest example. The CDQ-006 exemption was added to CDQ-006 without adding the definition of "root span" that would make it actionable. The ratio backstop was added to Transformation Rules as a guideline but never received a corresponding action clause.

The fix pattern is consistent across all five high-severity items: identify where the rule is stated most completely, make that the single authoritative statement, and remove or reduce the partial statements elsewhere. None of the fixes require changing what the rules require — only how precisely they are expressed.

The medium-severity ambiguities (COV-001 vs COV-004 boundary, COV-002 vs COV-006, notes format, RST-004 vs COV-004) share a different structural problem: **two rules that were designed independently were not reconciled when they intersected.** These require more design thought than the high-severity rewrites and are deferred to a future PRD per M3 scope constraints.
