- (2026-06-22) Fixed git auto-GC race condition in `test/helpers/git.ts`. Added `gc.auto = 0` and `maintenance.auto = false` to `makeTestRepo` so git never spawns background `pack-objects` processes during test cleanup. Without this, a second commit in a test could trigger auto-GC, which spawned a background process writing to `.git/objects/pack` while `afterEach` ran `rm({ recursive: true })` — causing an `ENOTEMPTY` failure when `rmdir` ran after Node.js had already emptied the pack directory but before the background process finished writing. The fix propagates to all test suites that call `makeTestRepo`. Fixes the `ENOTEMPTY: directory not empty, rmdir .git/objects/pack` failure in `test/git/per-file-commit.test.ts`.

- (2026-06-22) Added per-test diagnostic file writing to the coordinator acceptance gate (`test/coordinator/acceptance-gate.test.ts`). - (2026-06-22) Fixed two auto-fix gaps surfaced by taze run-16. (1) SCH-003 (`fixAttributeTypeCoercions`): added int-typed reverse case — when the schema declares `int` and the agent wrote `String(numericExpr)`, the wrapper is stripped to produce the correct numeric type. Covers both literal arguments (`String(42)`) and classified numeric expressions (`String(deps.length)`, `String(a - b)`). (2) CDQ-006 (`fixIsRecordingGuards`): removed the `isInsideEntryPoint` check from the auto-fix (Option A). The auto-fix and validator are now symmetric — both guard all expensive `setAttribute` calls regardless of the enclosing function's parameter names. Previously, functions with parameter names matching the framework-convention list (`req`, `res`, `ctx`, `context`, etc.) were silently skipped by the auto-fix while the validator still flagged them, causing unguarded calls to commit undetected in non-framework functions that happened to use those parameter names. Fixes issues #1010 and #1011.

- (2026-06-21) Expanded CDQ-007 (Attribute Data Quality) with a third sub-check for nullable member access: flags `setAttribute` calls where the value is a property access on a variable that may be null or undefined at the call site. - (2026-06-21) Fixed two SCH-001 false positive advisory sources. (1) The LLM judge namespace pre-filter now applies a secondary check for 2-component span names (e.g., `taze.workspace`): when the shared all-but-last prefix is a single token, the normalized last tokens must share a common prefix of length `max(2, floor(shorter.length / 2))` to pass to the judge. This prevents unrelated siblings like `taze.workspace` vs `taze.io` from reaching the judge while still comparing morphological variants like `taze.register` vs `taze.registration`. SCH-002 attribute keys skip this secondary filter. (2) The coordinator now strips SCH-001 non-blocking advisories after dispatch when the matched registry entry was introduced by an earlier file in the same run — a false positive caused by `dispatch.ts` calling `resolveSchema` fresh per file, causing each file to see previous files' span extensions as pre-existing registry entries. Updated `docs/rules-reference.md` SCH-001 entry to document both behaviors. Fixes issues #959 and #960.

- (2026-06-20) Extended `isExpectedConditionCatch()` in JS and TS COV-003 validators to recognize the negated ENOENT graceful-degradation pattern (`if (err.code !== 'ENOENT') throw err`). In this pattern ENOENT is the handled case (file simply doesn't exist) and non-ENOENT errors rethrow to an outer span that already records them — flagging the inner catch as needing error recording was a false positive that caused oscillation on file-reading operations. Both `err.code !== 'ENOENT'` and TypeScript cast forms are recognized. Updated `docs/rules-reference.md` COV-003 entry to document the new exception. Fixes issue #992.

- (2026-06-20) Added `fixUntypedStringMethods` reconciler to the fix-loop pre-validation pipeline. Detects `setAttribute` calls where a string method (`split`, `slice`, etc.) is called directly on a property-access expression (`obj.field.method()`) and wraps the receiver with `String(obj.field)` before each retry attempt — making CDQ-010 (untyped string method on property access) auto-correctable rather than requiring the agent to fix it through regeneration. Implemented in `cdq010.ts`, wired through `LanguageProvider` interface and both JS and TS providers, and called at both the main retry and advisory fix sites in `instrument-with-retry.ts`. Fixes issue #998.

- (2026-06-20) Added `fixDelimiterVariants` reconciler to the fix-loop pre-validation pipeline.
