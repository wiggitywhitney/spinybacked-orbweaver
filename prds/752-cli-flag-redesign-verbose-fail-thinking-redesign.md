## Problem

The current `--verbose` and `--thinking` flags have two usability gaps:

1. **Signal-to-noise**: `--verbose` emits the full structured block for every file, including correct skips that produce zero interesting output. In a 30-file run with 16 correct skips, the terminal is dominated by noise.

2. **Thinking inconsistency**: `--thinking` shows agent reasoning only for failed files, but advisory decisions on successful files are equally worth understanding during eval runs â€” the agent is already spending thinking tokens regardless.

3. **Ephemeral thinking blocks**: Thinking blocks disappear when the terminal session ends. If a user ran without `--thinking` and wants to review reasoning after the fact, the data is gone.

---

## Solution

**New flags:**
- `--verbose-fail` â€” structured per-file block for failed/partial files only; successful and skipped files show the compact one-liner
- `--thinking-fail` â€” thinking blocks for failed files only (replaces old `--thinking` behavior under a new, explicit name)

**Behavior change:**
- `--thinking` now shows thinking blocks for **all files** (not just failed), mirroring the `--verbose` convention: bare flag = all, `-fail` suffix = fail only

**Flag table after this PRD:**

| Flag | Output |
|------|--------|
| `--verbose` | Structured block, all files (unchanged) |
| `--verbose-fail` | Structured block, failed files only (new) |
| `--thinking` | Thinking blocks, all files (behavior change) |
| `--thinking-fail` | Thinking blocks, failed files only (new) |

**Companion files:** Thinking blocks are always written to `.instrumentation.md` companion reports when they exist, regardless of which display flags were passed. This preserves diagnostic data for post-hoc analysis even when no thinking flag was used.

---

## Context and Rationale

- **No backward compatibility concerns** â€” no external users at this stage.
- **`--thinking` behavior change is intentional** â€” consistency with `--verbose` convention matters more than preserving the current fail-only default. The `-fail` suffix makes fail-only explicit.
- **Thinking token cost** â€” thinking blocks are generated by the LLM regardless of display flags. Surfacing them costs nothing extra in API tokens; the only cost is terminal output and companion file size.
- **Advisory rules motivation** â€” during eval runs, advisory decisions (rules that don't fail files but could produce better results) are worth understanding. `--thinking` now surfaces that reasoning for every file.
- **Companion file rationale** â€” `.instrumentation.md` files are already ephemeral diagnostic artifacts committed to the instrument branch. Adding thinking blocks increases file size but provides durable access to reasoning that currently disappears with the terminal session.

## Design Notes

- The `--verbose` / `--verbose-fail` and `--thinking` / `--thinking-fail` pairs follow a consistent convention: bare flag = all files, `-fail` suffix = failed files only. No aliases (no `--verbose-all` or `--thinking-all`) â€” the bare flag is self-documenting through the help text.
- `--thinking` behavior change is a breaking change in the narrow sense, but backward compatibility is not a concern at this stage.
- The feature PR created by `/prd-done` needs the `run-acceptance` label to trigger acceptance gate CI. This is handled automatically by `/prd-done` when acceptance gate tests are detected.

## Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-05-04 | `--thinking` changes from fail-only to all-files | Mirrors `--verbose` convention; bare flag = all, `-fail` suffix = fail only. Consistency over backward compatibility (no external users). |
| 2026-05-04 | No `--verbose-all` / `--thinking-all` aliases | Aliases create confusion about whether they differ from the bare flag. Help text makes the "all" default explicit without extra flags. |
| 2026-05-04 | Thinking blocks always written to companion files | Companion files are diagnostic artifacts. Always writing thinking means data is never lost because the user forgot a flag. Cost is only file size. |
| 2026-05-04 | `--verbose-fail` applies to failed AND partial files | Partial files represent incomplete instrumentation â€” equally worth inspecting as full failures. |
